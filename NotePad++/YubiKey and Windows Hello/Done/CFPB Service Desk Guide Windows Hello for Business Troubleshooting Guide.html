CFPB End User Guide:

Windows Hello for Business Troubleshooting Guide

Overview

The Consumer Financial Protection Bureau (CFPB) has a requirement to provide phishing resistant Multi-Factor Authentication (MFA) for all users and computers. As a result, CFPB is using a multi-facet approach to MFA options provided. Windows Hello for Business (WHfB) replaces passwords with strong two-factor authentication on devices. The purpose of this document is to provide answers to frequently asked question related to troubleshooting WHfB enrollments and sign-in issues. It will be presented in a Knowledge Base (KB) article style by stating the problem, the cause, and the resolution. This document is intended primarily for Service Desk support staff but can be some items can be used for End User KBs as well.

Problem:

On the first-time sign-in to Windows 11 the user is not presented with the WHfB enrollment wizard.

Cause:

The user account is not associated with the correct group policy object in Active Directory, or the user account is not authenticating to Azure Active Directory during the sign-in process.

Resolution:

Things to check:

  •   First validate that the user account is in the '_GRP_Windows_Hello_Users' security group in
    Active Directory, if not add the user to the group.

  •   Make sure the person has a connection to the VPN if remote or CFPB LAN/WLAN if in the office, if not
    fix the connectivity issue.

  •   While logged in, open a command prompt in the user's context, not admin, then run the GPRESULT /R
    command. Validate that the following GPO is applied to the user's account, 'CLIENT UC WIN :
    Windows Hello for Business User Settings'    . If the GPO is not shown in the list, run the
    'GPUPDATE /FORCE' command and reboot. It is possible you may have to run this command more
    than once before the policy applies.

  •   While logged in as the user, open a command prompt in the user's context, not admin, then run the
    DSREGCMD /STATUS command. Validate that the command returns the value 'AzureADPrt: Yes'.

If the response is 'AzureADPrt: NO', escalate the ticket to Desktop Engineering for further assistance.

Problem:

After a successful enrollment, user is unable to login using WHfB. Error message: 'That option is temporarily unavailable. For now, please use a different method to sign in.'

Cause:

There is a synchronization that needs to occur between on-premises AD and AzureAD before the credential is available for use. By default, synchronization between Azure AD and On-premises AD require 2 hours.

Or

The computer is not applying the proper computer-based GPO.

Resolution:

  1.   Sign-in to the computer using another credential like the PIV card.

  2.   Open an elevated command prompt

  3.   Run the 'GPRESULT /R' command

  4.   Validate under the Computer Configuration that the following GPO is applied to the computer, 'CLIENT
    CC WIN : Windows Hello for Business Computer Settings    '.

  5.   If the GPO is not present, be sure you are connected to the VPN or LAN/WLAN and then run the
    'GPUPDATE /FORCE' command and reboot. Note it may take multiple attempts before the policy is
    applied.

  6.   If the above was done and the GPO is applied correctly then the issue may be related to the
    synchronization between AzureAD, and AD not being completed yet. If it has been within 2 hours of the
    enrollment the user may have to wait until the synchronization is complete. If the matter is urgent, it can
    be escalated to the Desktop Engineering or ICAM Engineering team to manually run the sync.

  7.   For Engineers, to run the sync.

    1.   Sign-in to the AD Connect server, WDCADCW01.cfpb.local

    2.   Open an elevated PowerShell prompt

    3.   Run the following command, 'Start-ADSyncSyncCycle-PolicyType Delta' to begin the
      synchronization. Synchronization when kicked off manually can take up to 15 minutes to complete.

Problem:

User forgets their WHfB PIN for sign-in

Cause:

PIN is unknown and user not able to sign-in using WHfB

Resolution:

If the user can sign in with a password, they can reset their PIN by selecting the "I forgot my PIN" link in Settings. For on-premises deployments, devices must be connected to the VPN or LAN/WLAN to access their on-premises network (domain controllers and/or certificate authority) to reset their PINs.

Reset PIN from Settings

  1.   Sign-in to Windows 10 using an alternate credential.

  2.   Open Settings, select 'Accounts ' > Sign-in options.

  3.   Select 'PIN (Windows Hello) ' > I forgot my PIN and follow the instructions.

Known limits of WHfB credentials

Windows Hello for Business cannot be used for login for the following applications:

  •   WebTA

  •   HR Connect

  •   Any application that requires PIV or PIV-D

  •   Mac laptops

  •   External websites
