      What are YubiKeys?

  • YubiKeys are phishing-resistant MFA hardware security tokens used for secure authentication

  • YubiKeys support Certificate Based Authentication and FIDO2 (Fast IDentity Online)/WebAuthN Authentication protocols

  • CFPB uses the 5C NFC FIPS USB-C model


    Benefits of using YubiKeys


    • Protects access to computers, networks, and online services

    • Easier and safer to use than authenticator applications

    • All-in-one configurable security key

    • Compatible with a variety of services and applications


    How do YubiKeys work?

    YubiKeys are simple to use:

    • For laptops you insert them into the USB port and tap the metal contact area when prompted and enter a PIN number


    • For mobile phones you tap the YubiKey to the NFC reader on the phone which is typically on the top back side of the phone

        Troubleshooting issues with YubiKeys

  • Issue: No security key icon on the sign in screen

  • Resolution: Verify that the GPO: CLIENT CC WIN : Security Key Login Setting is applied to the workstation.

    • Verify this by checking:

      1. Ensure that the registry key EnableFIDODeviceLogon is set to 1

        1. [HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\FIDO]

          1. "EnableFIDODeviceLogon"=dword:00000001

      1. [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Policies\PassportForWork\SecurityKey]

      2. "UseSecurityKeyForSignin"=dword:00000001


  • Issue: YubiKey locked from too many bad PIN attempts

  • Resolution: Reset the YubiKey and reenroll

    *See troubleshooting Guide for details


    Azure AD TAP (Temporary Access Password)

  • What is a TAP?

  • A Temporary Access Pass is a time-limited passcode that can be configured for single use or multiple. Users can sign in with a Temporary Access Pass to onboard other authentication methods including passwordless methods such as Microsoft Authenticator, FIDO2 or Windows Hello for Business.

  • A Temporary Access Pass also makes recovery easier when a user has lost or forgotten their strong authentication factor like a FIDO2 security key or Microsoft Authenticator app, but needs to sign in to register new strong authentication methods.

  • When to issue a TAP?

  • When a user does not have a PIV card or does not know their PIV PIN and needs to enroll into Windows Hello for Business and/or enroll a YubiKey.

    • This includes onboarding users and break fix users that doe not have PIV cards

    • Do not need to issue a TAP to a person if they have an active PIV and know their PIV PIN.

    • Do not need to issue a TAP to a person if they have an active YubiKey and know their PIN

    • Do not need to issue a TAP to a person if they are enrolled into Windows Hello for Business and know their PIN


      How to issue a TAP?

      1. Sign-in to the Azure portal by using your Azure AD ADM account.

      2. Select Azure Active Directory , browse to Users, select a user, then choose Authentication methods.

      3. If needed, select the option to Try the new user authentication methods experience.

      4. Select the option to Add authentication methods.

      5. Below Choose method, select Temporary Access Pass.

      6. Accept the default time or duration by selecting Add.

        • Once added, the details of the Temporary Access Pass are shown. Make a note of the actual Temporary Access Pass value. You provide this password to the user via Teams direct chat. The TAP will only show itself once, you can't view this value after you select Ok.

    Shipping Mode

  • The Shipping Mode screen is displayed after clicking "CONTINUE" on the "Build Completed Successfully Screen"

  • Shipping Mode is enabled by default with Windows 11 builds.

  • Applies a BitLocker boot pin – Securing the computer while in Transit.

  • Clicking "DISABLE" disables the bitlocker boot Pin.

  • Clicking "CONTINUE" keeps the boot bin enabled and will be displayed on next boot.


Shipping Mode Suggested Workflows


Hand Delivered Computers


  1. Build Completes

  2. Tech Clicks "DISABLE" on the Shipping Mode Screen

  3. Boot Pin Configuration is removed

  4. Device is handed over to user.

    Shipped Computers


    1. Build Completes

    2. Tech Clicks "CONTINUE" on the Shipping Mode Screen

    3. Tech Shuts powers off Laptop.

    4. Laptop is shipped to User

    5. Boot Pin is communicated to User proactively or when the user calls HD.

    6. User Enters Boot Pin, Computer Boots.

    7. User Clicks "DISABLE" on Shipping Mode Screen

    8. Boot Pin Configuration is removed.
