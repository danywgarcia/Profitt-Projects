Recovering a Drive in Recovery Mode

This article explains how to use the MBAM console to get a recovery password to give to an end user when their BitLocker-protected drive goes into recovery mode. Drives enter recovery mode if users lose or forget their PIN or password, or if the Trusted Module Platform (TPM) chip detects changes to the BIOS or startup files of a computer.

To get a recovery password, use the Drive Recovery area of the MBAM console.

Note: Users must be assigned either the MBAM Helpdesk Users or the MBAM Advanced Helpdesk Users role to access this area of the website. If you need but do not have these permissions, submit a Privileged User Access (PUA) Request.

Recovering a Drive

    1. Open a web browser and navigate to the MBAM Console.

      • MBAM Console for Windows 10 laptops.

      • MBAM Console for Windows 11 laptops.

    2. Select Drive Recovery in the left pane to open the Recover access to an encrypted drive page.

    3. Enter the end user’s Windows logon domain and username to view recovery information.

    1. Enter the first eight digits of the recovery Key ID to see a list of possible matching recovery keys, or enter the entire recovery key ID to get the exact recovery key.

Note: Service Desk must ask the user for the Key ID. The user will get the Key ID from the users’ device (screen).

    1. Select one of the predefined options from the Reason for Drive Unlock  list, then click Submit.

MBAM returns the following:

      • An error message if no matching recovery password is found.

      • Multiple possible matches if the user has multiple matching recovery passwords

      • The recovery password and recovery package for the submitted user

Note: If you are recovering a damaged drive, the recovery package option provides BitLocker with critical information that it needs to recover the drive.

    1. Click Copy Key to copy the password, then paste the recovery password into an email message. Alternatively, click Save to save the recovery password to a file.

Note: When the user types the recovery password into their machine or uses the recovery package, the drive unlocks.
